# Core Principles

These rules apply at ALL times during pSEO work — regardless of phase.

1. **AI never writes freeform.** Every generation fills a strict JSON schema with typed fields, enums, and quantity constraints.
2. **Niche context is everything.** If you swap the niche name and the content still makes sense, the context is too generic. Reject it.
3. **Content and presentation never mix.** JSON files contain data. Templates render it. These two layers stay separate.
4. **Titles are deterministic.** Templates like `"[N] [Type] for [Niche] in [Year]"` — never AI-generated.
5. **Quality test for every page:** "Would this be useful if search engines didn't exist?" and "Would someone bookmark this and come back?"
6. **External references are mandatory.** Every content JSON must include 3-5 outbound links to authoritative third-party sources. This signals credibility to both users and search engines. See `external-references.md`.
7. **Marketing-friendly tone.** Write for business owners and marketers, not developers. Use conversational language, short sentences, and concrete outcomes ("increase repeat purchases by 30%") instead of technical jargon. Every paragraph should answer "why should I care?" before "how does it work?". See `writing-style.md`.
8. **Contextual interlinking is mandatory.** Every page must include 2-4 inline links to other blog articles within body text (descriptions, context paragraphs). Links should feel natural — referencing related topics the reader might explore next. This boosts SEO (internal PageRank flow) and keeps readers on-site. See `interlinking.md`.
